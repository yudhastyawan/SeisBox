Coulomb Stress Analysis

Compute Coulomb Failure Stress changes (ΔCFF) caused by a source fault using the Okada (1992) dislocation model, with optional receiver fault optimization.

Overview

Coulomb Failure Stress (CFS) analysis quantifies how a mainshock (or any fault slip) redistributes stress in the surrounding crust, potentially promoting or inhibiting aftershocks or future earthquakes. SeisBox implements the complete Okada (1992) half-space dislocation model for ΔCFF computation with three operating modes.

Access from: Analysis → Coulomb Stress

Input File Format

Coulomb stress calculations require a Coulomb input file (compatible with the Coulomb 3.x format, .inp extension). This file specifies:

  • Elastic medium properties (Poisson's ratio, Young's modulus, friction coefficient)
  • One or more source fault segments (location, strike, dip, rake, slip, dimensions)
  • Output grid specification

Use the built-in Input Generator (accessible via the Open Input Generator button) to create or edit input files with a GUI form instead of writing them manually.

Operating Modes

Mode 1: Coulomb Grid

Computes ΔCFF on a regular 2D grid at a specified depth, for a given receiver fault orientation (strike, dip, rake). This is the most common use case — ideal for visualizing the spatial pattern of stress changes around the mainshock.

Mode 2: Deformation

Computes surface deformation (displacement vectors) caused by the source fault. Useful for cross-checking against InSAR or GPS data.

Mode 3: Batch Receiver

Evaluates ΔCFF at multiple receiver fault locations defined in a separate batch CSV file. Useful for computing stress changes at known aftershock hypocenters with their individual focal mechanisms.

Receiver Fault Parameters

For Coulomb Grid and Batch modes, you need to specify the geometry of the receiver fault (the fault plane on which stress change is evaluated):

ParameterDefaultDescription
Strike (°)30Strike direction of the receiver fault (0–360°, measured clockwise from North).
Dip (°)90Dip angle of the receiver fault (0–90°, 90° = vertical).
Rake (°)180Slip direction on the receiver fault (−180 to 180°). 0° = left-lateral, ±90° = dip-slip, 180° = right-lateral.

Grid Configuration

The calculation grid can be specified in two ways:

  • From input file: Use the grid bounds defined in your .inp file.
  • Override Grid: Check "Override Grid" to manually specify min/max X, Y (km from fault center) and grid spacing, or set geographic bounds (lon/lat with a reference origin point).

Depth Configuration

ParameterDescription
Depth (km)Single depth at which ΔCFF is computed (for non-range mode).
Depth RangeEnable to compute over multiple depth slices. Set Min Depth, Max Depth, and Depth Increment (all in km).

Receiver Fault Optimization

Enable "Optimize Receiver Fault" to search for the receiver fault orientation that maximizes the mean ΔCFF over a specified region. This is useful for identifying which fault geometries are most likely to be triggered by the source rupture.

ParameterDescription
Optimize Strike / Dip / RakeSelect which parameters to optimize. Each selected parameter is varied over the full geometric range.
Strike Increment (°)Step size for the strike search (e.g., 10° = 36 strike values tested).
Dip Increment (°)Step size for the dip search.
Rake Increment (°)Step size for the rake search.
Optimize for Shear / Normal / CoulombChoose which stress component to maximize.

Optimization with all three parameters and small increments (e.g., 5°) involves many thousands of model evaluations. Expect computation times of several minutes. Use coarser increments (e.g., 30°) for a rapid first pass.

Results & Visualization

Results are displayed as a 2D scatter plot (colored by ΔCFF value) overlaid on the mapped coastline. Color scale: red = positive ΔCFF (stress increase, promotes failure), blue = negative ΔCFF (stress shadow, inhibits failure).

Exporting Results

After calculation, use File → Export CSV to save the full grid results (X, Y, Depth, Shear, Normal, Coulomb columns) as a CSV file for further analysis in Python, MATLAB, or GIS software.
