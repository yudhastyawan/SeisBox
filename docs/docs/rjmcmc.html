RJ-MCMC HVSR Inversion

Bayesian Reversible-Jump Markov Chain Monte Carlo inversion to estimate S-wave velocity structure from an observed HVSR curve.

Overview

The RJ-MCMC method treats the number of subsurface layers as an unknown parameter. Rather than fixing the number of layers, the algorithm explores models with variable dimensionality, automatically identifying the optimal model complexity supported by the data. The result is a probabilistic ensemble of Vs(z) profiles (posterior distribution) rather than a single deterministic model.

Access this module from: Analysis → RJ-MCMC HVSR Inversion

Input Requirements

The inversion requires a CSV file containing the observed HVSR curve. This is typically the output from the HVSR Analysis module.

Expected CSV format:

Frequency,Mean_HV,Std_Dev_Plus,Std_Dev_Minus
0.5000,1.2341,1.5102,0.9871
1.0000,2.1500,2.4000,1.9000
...

The inversion uses the HVf binary (bundled inside SeisBox) to compute the synthetic HVSR forward model. Its path is auto-detected from the application bundle. If not found automatically, use the Browse button to locate it manually.

Parameters

MCMC Parameters

ParameterDefaultDescription
Total Iterations100,000Total number of MCMC steps. Higher = better posterior exploration, but slower. Minimum recommended: 50,000.
Burn-in20,000Number of initial iterations discarded as the chain converges. Should be at least 20% of total iterations.
Thinning100Every n-th sample is saved. Reduces auto-correlation in the posterior chain. Output samples = (total − burnin) / thin.
Initial Search Attempts10Number of random starting models tried before the main chain. The best starting model is selected.

Prior Boundaries

ParameterDefaultDescription
Vs Range (m/s)100 – 2000Uniform prior bounds on S-wave velocity for each layer. Set based on your expected geological context.
Thickness Range (m)5 – 100Uniform prior bounds on individual layer thickness.
Layer Count3 – 10Allowed range for the number of layers. The algorithm will explore this range automatically.
Total Depth (m)10 – 300Constraint on the total depth of the model. Models whose cumulative thickness falls outside this range are rejected.

Structural Probabilities

ParameterDefaultDescription
Prob Ascending Vs0.8Probability that a proposed new Vs is higher than the layer above (enforces velocity increasing with depth). Set to 0 to allow any order.
Prob Ascending Thickness0.0Similar constraint on layer thickness. 0 = unconstrained.

f0 / A0 Target Misfit

Optionally penalize models whose synthetic peak frequency (f0) or peak amplitude (A0) falls outside the specified range.

ParameterDefaultDescription
Target f0 Range (Hz)1.0 – 2.0Models with synthetic f0 outside this range receive an additional misfit penalty.
f0 Weight0.0Weight of the f0 misfit term in the total cost function. 0 = disabled.
A0 Weight0.0Weight of the A0 (peak amplitude) misfit term. 0 = disabled.

Time-Averaged Vs Constraint

Enable this option to constrain the time-averaged S-wave velocity (Vsz) at a specified depth, which is useful when an independent estimate (e.g., from downhole measurements) is available.

ParameterDescription
Target Depth (m)Depth at which the time-averaged Vs is computed (e.g., 30 m for Vs30).
Avg Vs Range (m/s)Acceptable range for the time-averaged Vs at that depth. Models outside are rejected.

Running the Inversion

  1. Set Observation CSV

    Browse to or type the path of your HVSR CSV output from the HVSR Analysis module.

  2. Set Output File

    Choose a .jsonl output path where the posterior chain samples will be saved.

  3. Configure Parameters

    Adjust MCMC, prior, and structural parameters as appropriate for your site.

  4. Run

    Click "Generate Config & Run Inversion". The terminal log panel shows real-time iteration output. The button is disabled while running.

A full run of 100,000 iterations typically takes 1–5 minutes depending on the number of frequency points and CPU speed.

Visualizing Results

After the inversion completes (or if you have a previous .jsonl output), click "📂 Load & Visualize JSONL Output" and select your output file. The visualizer requires the observation CSV to be set.

Four diagnostic plots are displayed:

  • Posterior HVSR Fits: Synthetic HVSR curves from all posterior samples, color-coded by RMSE (orange = higher misfit, yellow = lower). Black dots = observed data.
  • Posterior Vs Profiles: All accepted Vs(z) profiles. Thick blue = median; light blue dashed = P05/P95 credible interval; black = best-fit (lowest RMSE).
  • RMSE Histogram: Distribution of model misfits.
  • RMSE vs Index & Layers vs Index: Convergence diagnostics.

Geotechnical Parameters Summary

A summary panel shows the posterior estimates (mean and range) of standard geotechnical parameters:

  • Vs30: Time-averaged Vs at 30 m depth.
  • H800: Depth to Vs = 800 m/s.
  • Z1.0: Depth to Vs = 1000 m/s.
  • Z2.5: Depth to Vs = 2500 m/s.
