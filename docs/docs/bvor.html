BVor — Bayesian Voronoi Analysis

Detect spatial or temporal changes in seismicity rates (b-value) using a transdimensional Bayesian Voronoi tessellation framework.

Overview

The BVor module implements a Bayesian non-parametric method for mapping the Gutenberg-Richter b-value across space (latitude-longitude) or time-depth domains, using a Voronoi tessellation that adapts to the data distribution. The number of Voronoi cells (nodes) is treated as an unknown and estimated from the data, avoiding overfitting.

Access from: Analysis → B-Value Voronoi Analysis

Analysis Modes

  • Spatial: Maps b-value as a function of geographic location (longitude, latitude). Reveals spatial heterogeneity in seismic hazard or fault complexity.
  • Temporal: Maps b-value as a function of normalized time and depth. Reveals temporal evolution of seismicity (e.g., b-value changes before/after a mainshock).

Input Data

You can provide data in two ways:

  • From CSV file: Load any earthquake catalog CSV with columns for lon, lat, mag, depth, and a time/date column (RFC3339 format recommended).
  • From active Spatial Map: Leave the CSV field empty to use the earthquake events currently loaded in the ISC Catalog Search / Spatial Analysis window.

Parameters

Voronoi Node Configuration

ParameterDefaultDescription
N Nodes Min2Minimum number of Voronoi cells explored. Must be ≥ 2.
N Nodes Max60Maximum number of Voronoi cells. Larger values allow more spatial detail but increase computation time.
Grid Resolution200Size of the output interpolation grid (200 = 200×200 pixels). Higher values give smoother output maps but take longer.
Min Obs/Cell5Minimum number of earthquakes required in a Voronoi cell for b-value estimation. Cells with fewer events are excluded.

Initialization Methods (Realizations per N)

For each number of nodes N in [N_min, N_max], the algorithm runs multiple Voronoi realizations initialized using different strategies. The best-fitting realization (by BIC) is selected.

MethodDefaultDescription
Sobol30Quasi-random Sobol sequence initial node placement — good coverage of the domain.
Uniform30Uniformly random node placement.
Data30Nodes placed at random data points — naturally clusters where seismicity is dense.
KDE30Nodes sampled from a Kernel Density Estimate of the data distribution — more focused in high-density regions.
K-Means1K-means clustering of data points for initial node placement (deterministic, only 1 realization needed).

Computational Resources

ParameterDescription
CPU CoresNumber of parallel threads used. Defaults to the number of logical CPU cores available. All node counts are evaluated in parallel.

Running the Analysis

  1. Select Mode

    Choose Spatial or Temporal depending on your research question.

  2. Load Data

    Browse to a catalog CSV or ensure events are loaded in the Spatial Map.

  3. Set Output File

    Choose a .npz output file path (NumPy archive format).

  4. Click "Run B-Value Analysis"

    The progress bar and log panel on the right show real-time progress across all N-node configurations.

  5. Visualize

    Click "Visualize Results" to open the BVor visualization window.

The total realizations = (N_max − N_min + 1) × sum(init_methods). For default settings: 59 × 121 ≈ 7,139 Voronoi tessellations evaluated. Use fewer realizations per method or a smaller N range for faster exploratory runs.

Visualization

After loading the .npz output in the Visualization window, you can inspect:

  • b-value map: Color-coded spatial/temporal distribution of the Gutenberg-Richter b-value.
  • BIC curve: Bayesian Information Criterion vs. number of nodes — the optimal model corresponds to the BIC minimum.
  • Error/uncertainty map: Spatial distribution of b-value estimation uncertainty.

Output File Format

The .npz output is a NumPy archive file readable in Python:

import numpy as np
data = np.load("bvalue.npz")
print(list(data.keys()))  # e.g. ['b_grid', 'x_grid', 'y_grid', 'bic', ...]
