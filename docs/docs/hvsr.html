HVSR Microtremor Analysis

Compute the Horizontal-to-Vertical Spectral Ratio (HVSR) from three-component ambient noise records following the SESAME 2004 guidelines.

Overview

HVSR (also called H/V ratio or Nakamura method) is a widely used technique for estimating the fundamental resonance frequency (f0) of soft-soil sites and inferring shallow velocity structure. SeisBox implements the full SESAME-compliant processing pipeline: windowing → anti-trigger screening (STA/LTA) → FFT → Konno-Ohmachi smoothing → spectral ratio computation → iterative f0 filtering → SESAME criteria evaluation.

Prerequisites

Before opening the HVSR window, load your three-component waveforms (Z, N/1, E/2) through the Waveform Viewer. SeisBox automatically detects and assigns components based on channel codes (e.g., BHZ, BHN, BHE). Linear trend removal (detrending) is applied automatically.

Access HVSR Analysis from: Analysis → HVSR Microtremor Analysis

Processing Workflow

  1. Set Parameters

    Configure all parameters in the left panel (see tables below).

  2. Preview Windows (Step 1)

    Click "1. Preview Windows (STA/LTA)" to segment the record and visualize which time windows pass the anti-trigger filter. Valid windows are shown in green, rejected ones in red on the time-domain plot. You can click on any window in the plot to manually toggle its valid/invalid status.

  3. Process HVSR (Step 2)

    Click "2. Process HVSR" to run the full processing pipeline in a background thread. A progress bar shows completion. Results appear as three sub-plots (Stages 1–3) once done.

  4. Inspect Results

    Review the SESAME criteria panel and the three-stage HVSR curve plots (Raw → STA/LTA filtered → f0 filtered).

  5. Export

    Click "💾 Export CSV" to save the HVSR curve (Frequency, Mean_HV, Std_Dev_Plus, Std_Dev_Minus).

Parameters

Windowing & Tapering

ParameterDefaultDescription
Window Length (s)20Length of each time window in seconds. Longer windows give better frequency resolution at low frequencies. SESAME recommends at least 10× the longest period of interest.
Overlap (%)50Overlap between consecutive windows. 0–90%. Higher overlap increases the number of windows (reduces variance) but introduces correlation.

Anti-Trigger (STA/LTA)

Windows contaminated by transient noise (vehicles, footsteps, wind gusts) are rejected using the Short-Term Average / Long-Term Average (STA/LTA) ratio computed on the Z component.

ParameterDefaultDescription
STA Length (s)1.0Duration of the short-time averaging window.
LTA Length (s)30.0Duration of the long-time averaging window.
Threshold T1 (Lower)0.1Windows where any sample has STA/LTA < T1 are rejected (stationary requirement).
Threshold T2 (Upper)3.0Windows where any sample has STA/LTA > T2 are rejected (transient rejection).

A good starting point for urban environments: STA=1s, LTA=30s, T1=0.1, T2=2.5. For very noisy sites, relax T2 to 3.0–5.0.

Smoothing & Frequency

ParameterDefaultDescription
Konno-Ohmachi (b)40Smoothing bandwidth coefficient. Higher values = narrower smoothing. SESAME recommends b=40. Typical range: 10–100.
Min Frequency (Hz)0.5Minimum frequency displayed and used in SESAME criteria.
Max Frequency (Hz)20.0Maximum frequency.
Sample Count200Number of logarithmically spaced frequency points between Min and Max.
Horizontal CombineGeometric MeanMethod to combine N and E spectral amplitudes before dividing by Z. Options: Geometric Mean, Quadratic Mean (RMS), Arithmetic Mean, Maximum.

f0 Iterative Filter

After STA/LTA screening, an optional second-pass filter removes windows whose individual peak frequency (f0) deviates too far from the ensemble mean.

ParameterDefaultDescription
Enable Iterative f0 FilterOffEnable to activate the second filtering stage. Recommended for sites with multi-modal H/V curves or strong local interference.
Multiplier (n)1.5Windows whose f0 falls outside [mean − n·σ, mean + n·σ] are rejected. Smaller n = stricter filtering.

Result Plots

After processing, three HVSR curve stages are displayed stacked vertically:

  • Stage 1: Raw (All Windows) — HVSR from all windows without filtering.
  • Stage 2: STA/LTA Filtered — Only windows that passed the anti-trigger check.
  • Stage 3: f0 Filtered — (if enabled) Windows further filtered by f0 consistency.

In each plot: thin grey lines = individual window HVSR curves; thick blue line = mean HVSR; dashed blue = ±1σ; red diamond + dashed line = peak frequency f0; green lines = mean f0 ± σ.

SESAME Criteria

SeisBox evaluates the official SESAME 2004 reliability and clear-peak criteria automatically and displays a pass/fail (✅/❌) summary for all 9 conditions:

Reliability Criteria (3)

  • C1: f0 > 10 / lw (f0 must be resolved by the window length)
  • C2: nc(f0) > 200 (sufficient cycles of the fundamental frequency)
  • C3: σA(f) < threshold for 50% of the frequency range

Clear Peak Criteria (6)

  • C1: There exists f− ∈ [f0/4, f0] with A(f−) < A0/2
  • C2: There exists f+ ∈ [f0, 4f0] with A(f+) < A0/2
  • C3: A0 > 2 (peak amplitude at least 2)
  • C4: f0 peaks from individual windows within ±5% of mean f0
  • C5: σf < ε(f0)
  • C6: σA(f0) < θ(f0)

A minimum of 5 out of 6 clear-peak criteria must pass for the curve to be considered a reliable peak.
